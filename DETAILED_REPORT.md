## 🔧 技术栈

### 后端技术
- **语言**: Python 3.12
- **框架**: Streamlit (Web UI)
- **数据库**: SQLite (通过SQLAlchemy)
- **核心库**:
  - `hashlib` - MD5哈希计算
  - `pathlib` - 路径处理
  - `zipfile` - 备份压缩
  - `logging` - 日志记录
  - `json` - 数据存储
  - `re` - 正则表达式

### 前端技术
- **框架**: Streamlit
- **图表**: Streamlit原生图表 (st.line_chart, st.bar_chart)
- **组件**: Streamlit原生组件 (st.metric, st.progress, st.expander)
- **样式**: 自定义CSS

### 开发工具
- **代码审查**: 手动审查
- **版本控制**: Git
- **文档**: Markdown

## ✅ 具体功能实现

### Backend模块 (4个)

| 模块 | 功能 | 技术细节 |
|:---|:---|:---|
| **cache_manager.py** | MD5缓存系统 | hashlib.md5, 分级存储(cache/xx/xx.json), 30天过期 |
| **file_utils.py** | 文件安全工具 | 路径遍历防护, PDF魔术字节验证, 文件名清理 |
| **backup_manager.py** | 备份管理系统 | zipfile压缩, 自动备份配置, 保留策略 |
| **security.py** | 安全功能 | 密码强度算法(0-100分), 登录锁定(5次/30分钟) |

### Frontend组件 (3个)

| 组件 | 功能 | 技术细节 |
|:---|:---|:---|
| **dashboard.py** | 仪表盘 | 4个st.metric卡片, 趋势图(st.line_chart), 饼图(st.bar_chart) |
| **progress.py** | 进度组件 | 实时进度条, 速度计算, 预计时间, 批量追踪器 |
| **guide.py** | 新手引导 | st.expander, 3步流程, 首次登录检测, session_state |

### 系统集成

| 集成点 | 功能 | 技术细节 |
|:---|:---|:---|
| **main.py** | 主程序集成 | 导入所有模块, 仪表盘路由, 7页面菜单 |
| **缓存集成** | 解析加速 | compute_md5 + is_cache_valid + cache_result |
| **安全集成** | 密码/锁定 | check_password_strength, record_login_attempt |
| **备份集成** | 数据保护 | create_backup, restore_backup, auto_backup |

## 📚 经验教训（供下次参考）

### 人员安排经验

1. **OpenCode** - 适合：
   - ✅ 基础工具开发
   - ✅ 需要快速完成的任务
   - ✅ 对稳定性要求高的模块
   - ❌ 不适合：复杂UI设计

2. **Kimi Code** - 适合：
   - ✅ 前端界面开发
   - ✅ 用户体验优化
   - ✅ 需要详细文档的任务
   - ❌ 不适合：底层架构

3. **Cursor Composer** - 适合：
   - ✅ 系统集成
   - ✅ 架构设计
   - ✅ 复杂多模块协调
   - ❌ 不适合：紧急快速任务

### 工作流程改进

1. **任务分配前**：
   - 制定详细开发规范
   - 明确文件保存位置（绝对路径）
   - 提供参考代码示例

2. **开发过程中**：
   - 不催促，给充足时间
   - 回答开发者问题
   - 不干预具体实现

3. **完成后**：
   - 自己先全面审查
   - 检查架构符合性
   - 验证文件实际存在
   - 测试代码可运行
   - 确认100%后再汇报

4. **汇报时**：
   - 按模板详细汇报
   - 包含技术栈、功能、人员表现
   - 总结经验教训
